# GoTreeSitter Without CGo — a real-lane deck

A full conference talk built on the gosx-slides **real lane**: live GoSX islands,
evaluated `{expr}`, syntax-highlighted code with click-through stepping, themes,
speaker notes, and a synced presenter view. It ports the "GoTreeSitter Without
CGo" talk (the pure-Go tree-sitter story) into a deck that *runs what it
describes* — the benchmark and the syntax tree are real, hydrated components, not
screenshots.

## Run it

```sh
slides serve examples/gotreesitter          # http://127.0.0.1:8080
slides serve examples/gotreesitter --watch  # + hot-swap: edit a .gsx, it swaps live
```

Then: `→` / `Space` to advance · `o` overview grid · `p` presenter view (current +
next + notes + timer, synced peer-to-peer) · `f` fullscreen. The three code slides
**step** through one reveal at a time before advancing; the two charts are live —
click them.

## What's in here

| File | Role |
| --- | --- |
| `deck.md` | The 14-slide deck (theme `neon`). Headmatter picks the theme; per-slide ` ```yaml ` fences set `layout:`. |
| `Benchmark.gsx` | Live island — the `5.95x → 4.41x` GLR fork-reduction bar chart. "Collapse forks" / "Reset". |

`build/` (runtime.wasm + bootstrap, generated by `serve`) is gitignored.

## Features this deck exercises

- **Live islands** — `<Benchmark/>` and `<ParseTree/>` compile to island bytecode and hydrate in the browser.
- **Code stepping** — the methodology / safety-criterion / transfer-limit slides use ` ```go {1-2|4-6|8} ` fence meta; each `|` group is one reveal step the arrow keys walk through.
- **Evaluated expressions, themes + web-fonts, layouts, speaker notes, overview, presenter view** — all from deck source.

## Authoring islands

An island is `package main` + `//gosx:island` + `func Name(props any) Node { … }`
beside `deck.md`; reference it as `<Name Prop={value}/>`. Hold state in
`signal.New(v)` and read it as `{s.Get()}` in text or inside `style={"…" + s.Get()}`;
`onClick={handler}` wires events. Props bind by **exact attribute name**
(`<Benchmark Before={5.95}/>` → `props.Before`). See `Benchmark.gsx` and
`ParseTree.gsx` for worked examples.

Deck-authoring conventions: slide separators are blank-line-surrounded `---`;
per-slide frontmatter is a leading ` ```yaml ` fence, not a `---` block.
